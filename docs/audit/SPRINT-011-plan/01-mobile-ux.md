# Mobile UX Audit — 2026-04-20 prod

Persona: one-handed right-thumb dispatcher on a scooter, +35 °C, 20 с/задача, iPhone 14 PWA.
Source: prod screenshots 01–13 in `docs/audit/screens/` + code review of `components/layout`, `components/calendar`, `app/dashboard/*`.

---

## Critical (ship-blockers or daily friction)

- **[C1] Dashboard week — нет ни одной «+ Запись» кнопки в thumb-zone.** Новая запись создаётся тапом по пустому слоту, но на +35 °C с потной рукой тап попадает на 09:00 ПН (верх экрана, физически недостижим большим пальцем при хвате одной рукой). «+ Расход» есть, а «+ Запись» — нет. **Fix (S):** FAB 56×56 `bottom-[86px] right-4` (над BottomTabBar) с плюсом → open AppointmentSheet. `app/dashboard/page.tsx`.
- **[C2] BottomTabBar — нет центральной кнопки Создать.** Центр экрана (самое лёгкое место для большого пальца) занят вкладкой «Чаты». Primary action (новая запись) вытеснен. **Fix (M):** заменить/вставить круглую центральную «+» кнопку (паттерн Instagram/AirFix) — уже заложено в комментарии `BottomTabBar.tsx:17-19` «centre + Запись pill» но не реализовано. `components/layout/BottomTabBar.tsx`.
- **[C3] Week view — колонки нечитаемы.** 7 дней / 390 px = 47 px на колонку; внутри колонки метка «ПАФОС ПН 20» в 3 строки шрифтом 11 px. Прочитать на солнце за 1 с — нельзя. **Fix (M):** вернуть `3days` как дефолт на мобиле, week только ландшафт; или уменьшить «ПАФОС» в подзаголовок и оставить «ПН 20» крупно. `components/calendar/WeekView.tsx`, `DayColumn.tsx`.
- **[C4] Day view full-height header занимает 25% viewport под «ПАФОС ПН 20» без единой записи.** Диспетчер смотрит план — видит пустой синий прямоугольник на полэкрана. **Fix (S):** высота day-header 48 px max, фон тоньше. `DayColumn.tsx`.
- **[C5] Clients page — PWA «Установить Babun CRM» баннер перекрывает BottomTabBar.** Нижняя вкладка «Клиенты» в скриншоте prod-04 обрезана — tab bar не виден, пользователь теряется. **Fix (S):** баннер должен иметь `mb-[62px]` или закрываться по свайпу вниз. `components/pwa/*` и `app/dashboard/layout.tsx`.
- **[C6] Purple header 106 px — 12% экрана, содержит только «Напоминания / Финансы / Настройки / Расходы / Зарплата / Отчёты».** На subpage отбирает скролл-область у контента. **Fix (S):** схлопнуть в 44-48 px single-row. `components/layout/PageHeader.tsx`.
- **[C7] Tabs «Y&D / D&K» на dashboard под хедером выглядят как горизонтальный скролл, но это и есть фильтр бригад.** Надпись `Y&D` непонятна стороннему пользователю (это инициалы мастеров). Для дежурного на скутере ок, для SaaS — провал. **Fix (M):** показать иконку/цвет бригады + сокращение, не только инициалы. `components/layout/Header.tsx:230-278`.

## Important (noticeable annoyance)

- **[I1] Header hamburger в top-left** — 390 px phone, top-left = самая недоступная точка для правши с большим пальцем. `Header.tsx:71-82`. Уже есть «Ещё» в нижней панели — рассмотреть удаление гамбургера на мобиле. **Fix (S).**
- **[I2] Today-icon (calendar с числом 20) 36×36 px** — ниже 44 px HIG. `Header.tsx:125`. **Fix (S):** `w-11 h-11`.
- **[I3] Dropdown «View mode» 120 px min-width** — открывается в правом углу, может обрезаться safe-area notch на некоторых iPhone. Список Day/3дня/Неделя/Месяц — можно заменить на bottom sheet. `Header.tsx:179`. **Fix (M).**
- **[I4] Finances — 4 stat-карточки ДОХОД/РАСХОД/ПРИБЫЛЬ/ДОЛГИ** в ряду 85 px каждая. Цифры `€3 550` занимают всю ширину, плюс `+100%`. В следующем спринте при больших суммах (`€35 000`) сломается. **Fix (S):** 2×2 grid или горизонтальный scroll.
- **[I5] Finances — ТАБЫ «Все / Y&D / D&K»** дублируются дважды: один раз как табы бригад, второй раз те же буквы в reports page и payroll. Cognitive load. **Fix (S):** одна панель выбора бригад глобально.
- **[I6] Reports — таблица дат** `04-16 / +€300 / — / +€300` — формат `04-16` нечитаем (это MM-DD а не DD.MM к которому привыкли в Европе). Длинная колонка `РАСХОД` показывает `—` на каждой строке (пусто). **Fix (S):** формат `16 апр`, скрыть пустую колонку расходов. `app/dashboard/reports/`.
- **[I7] Payroll — внизу строка `Закрыть неделю и создать период` уходит под BottomTabBar.** Текст читается только наполовину. **Fix (S):** `pb-[86px]` на контейнере. `app/dashboard/payroll/`.
- **[I8] Expenses — CTA «Засеять демо-данные»** висит на prod! Пользователь увидит это как основную функцию. **Fix (S):** убрать из прода через `process.env.NEXT_PUBLIC_DEMO === "1"` guard.
- **[I9] Waitlist — горизонтальный scroll табов «Все / Ожидает / Связались / Записан / Не актуально»** последний таб обрезается (`Не актуальн`). Нет визуального fade/arrow indicator что есть ещё контент. **Fix (S):** `overflow-x-auto` + градиент fade справа.
- **[I10] Chats — табы «Все / Без ответа / WhatsApp / Instagram / Telegram(` — обрезается число.** То же что [I9], плюс «Без ответа» подсвечен оранжевым — хороший visual cue, но только в одном табе. **Fix (S).**
- **[I11] Chat rows — `«У нас 4 кондиционера, все нужно почистить»` — текст серым `slate-500`** на белом + время `1ч 1м` в двух версиях (часы + минуты). Двойное время = шум. **Fix (S):** одна метка «1 ч».
- **[I12] Settings — 6 строк по 64 px: «Календарь / Города / Финансовые бригады / Расписание команд / Шаблоны SMS / Услуги и категории»** — первый экран показывает только 5.5 строк из ~12. Диспетчер не знает, что ниже. **Fix (S):** визуальный cue «scroll for more» или icon-grid 2 cols.
- **[I13] Settings rows — подписи `slate-400` на `slate-50`** ("Справочник городов для клиентов и записей") — ниже contrast floor. Lighthouse тоже жалуется (color-contrast fail). **Fix (S):** `slate-500`. `app/dashboard/settings/page.tsx`.

## Minor (polish, not urgent)

- **[M1] Week view времена 08:00–14:00** на скриншоте — т.е. пользователь видит 6 слотов из 12+ рабочих часов. Не хватает индикатора «листай вниз». **Fix (S):** subtle scroll-hint fade.
- **[M2] Recurring page empty-state** — хороший визуал, но текст «После выполненной записи … → «Повторить через…»» с unicode «…» и стрелкой `→` выглядит как placeholder. **Fix (S):** переписать: «Отметь запись выполненной и выбери «Напомнить через X дней» — карточка появится здесь.»
- **[M3] Chat unread dot** зелёный (на М/К/«1» в кружке) — противоречит красной точке на табе «Чаты (2)». Два цвета для одного смысла. **Fix (S):** один цвет.
- **[M4] Dashboard «2» badge на «Чаты» в BottomTabBar** не обведён `ring-2 ring-white` — сливается со всем. `BottomTabBar.tsx:140` кажется правильным, но на скриншоте ring еле виден. **Fix (S):** увеличить ring до 2.5 px.
- **[M5] Finance card «ПРИБЫЛЬ» с активной фиолетовой рамкой** — при этом ни одна карточка не interactable. Фейковая affordance. **Fix (S):** убрать ring, сделать карточки кликабельными (ведут на детализацию).
- **[M6] «+ Расход» pill на календаре** — это единственная создающая кнопка, но без иконки кошелька. На солнце «+ Расход» без иконки = 2 с на распознавание. **Fix (S):** добавить lucide `Receipt`.

3. **Убрать «Засеять демо-данные» с production** через env-guard. [I8]
4. **MM-DD → DD MMM** в reports table. [I6]
5. **slate-400 → slate-500** в Settings подписях — чинит Lighthouse a11y contrast. [I13]
6. **pb-[86px]** на payroll, expenses, reports, recurring containers. [I7]
7. **Увеличить today-кнопку и view-mode кнопку до 44×44.** [I2]

## Summary

**Вердикт: fix before ship.** 3 ship-blocker-а ([C1] нет primary action в thumb-zone, [C2] центр BottomTabBar потерян, [C3] week view не читается на 390px). Финансы/Payroll/Reports/Settings — функционально есть, визуально почти готовы, но без `pb-[86px]` нижние CTA режутся, а `slate-400` текст на Cyprus-солнце не прочитать. Quick-wins (8 пунктов) закрываются за ~3 часа и поднимают ежедневный comfort радикально.

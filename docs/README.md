# Babun docs/ — orientation

Map of what lives here and when to read what.

## Reference docs (read these first)
- **`../CLAUDE.md`** — project conventions, Golden Rules, Critical Known Issues. Loaded into every Claude Code session.
- **`architecture.md`** — current Babun architecture.
- **`coding-patterns.md`** — code style and patterns.
- **`roadmap.md`** — what we're building next.
- **`design-language.md`** — visual tokens, spacing, typography.
- **`finance-model.md`** — money math, currency, rounding rules.
- **`agents.md`** — overview of `.claude/agents/`.

## Autopilot infrastructure (Phase-built)
- **`AUTOPILOT_DEVIATIONS.md`** — every place this autopilot setup deviated from the literal plan, with the babun-specific reason. **Always start here when something looks off.**
- **`_story-template.md`** — canonical STORY skeleton. Strategist fills `## Why now / Scope / Acceptance`, architect appends `## Data model / File plan / Test plan / Rollback`, downstream agents append their pipeline log.
- **`BACKLOG.md`** — auto-generated by `/audit-all-pages`. 64 routes scored 0–20 on functionality / UI / perf / security. Re-run when Sentry MCP is OAuth-authenticated to enrich scores.
- **`RLS_AUDIT.md`** — Supabase tenant-isolation state, table-by-table. Snapshot of what already exists; cross-tenant probe coverage status.
- **`SETUP_CHECKLIST.md`** — manual steps the autopilot can't do for you (GitHub Secrets, Vercel env, MCP OAuth, Telegram bot, `/schedule` Routine, `/install-github-app`).

## Subdirectories
- **`adr/`** — Architecture Decision Records (one file per significant decision).
- **`audit/`** — older audit notes.
- **`corporation/`** — business-side docs.
- **`ideas/`** — capture-bucket; not stories.
- **`sprints/`** — sprint plans.
- **`stories/`** — `STORY-NNN.md` and `STORY-NNN-<slug>.md`. Every autopilot run writes its own story here.

## Commands at a glance

### Manual mode (existing)
| Command | What it does |
|---|---|
| `/plan {feature}` | Strategist drafts `docs/stories/STORY-NNN.md`. Always run before code. |
| `/implement {story}` | Developer implements an approved story. |
| `/test` | tsc + eslint (+ vitest if present). |
| `/review` | Code review of `git diff master`. |
| `/status` | Snapshot of project state. |
| `/debug {desc}` | Diagnose a bug; fix root cause. |
| `/setup` | Verify dev environment. |
| `/full-pipeline {arg}` | 9-phase babun pipeline using chrome-devtools MCP. Existing. |

### Autopilot mode (new, this setup)
| Command | What it does |
|---|---|
| `/audit-all-pages` | Re-scan every route under `babun-crm/apps/web/src/app/`, regenerate `BACKLOG.md`. |
| `/full-pipeline-autopilot [--max-pages=N] [--page=/route]` | State-machine: strategist → architect → developer → tester → designer → security-auditor → performance-optimizer per page. Default N=1. |
| `/polish-page <route>` | Run only designer + perf + security on an existing page. |
| `/fix-all-bugs` | Pull top Sentry issues (≤ 5) and dispatch a developer per issue. |
| `/production-ready` | Final pre-ship gate; exits non-zero if any axis fails. |

## The pipeline state markers

The autopilot is driven by exact strings that each subagent emits on its last line:

```
READY_FOR_ARCH: STORY-NNN     ← strategist
READY_FOR_BUILD: STORY-NNN    ← architect
READY_FOR_TEST: STORY-NNN     ← developer
READY_FOR_DESIGN: STORY-NNN   ← tester (green)
BUGS_FOUND: STORY-NNN         ← tester (red, bounces to developer; max 2 retries)
READY_FOR_SECURITY: STORY-NNN ← designer
READY_FOR_PERF: STORY-NNN     ← security-auditor (clean)
SECURITY_BLOCK: STORY-NNN     ← security-auditor (red; bounces to developer)
READY_TO_MERGE: STORY-NNN     ← performance-optimizer (green)
PERF_BLOCK: STORY-NNN         ← performance-optimizer (red)
STOP: <reason>                ← any agent's kill switch (halts loop, pings Telegram)
```

Three consecutive bounces for the same STORY also halt the loop.

## How to run for the first time

1. Read `SETUP_CHECKLIST.md` and tick every box. Most are one-time external setup (GitHub Secrets, Vercel bypass, MCP OAuth).
2. Open a fresh Claude Code session and let it boot — `SessionStart` hooks inject branch + last story + last CI status automatically.
3. Run `/audit-all-pages` once and skim `BACKLOG.md` to confirm the top-5 still match what you expect.
4. Pick the recommended first page (`/dashboard/sms-templates`) and run `/full-pipeline-autopilot --page=/dashboard/sms-templates` **interactively** so you can see the state markers light up. Stop on the first surprise; tune prompts.
5. Once you're comfortable, set up the cloud Routine: `/schedule daily babun autopilot at 06:00 Europe/Nicosia` with prompt `Run /full-pipeline-autopilot --max-pages=3`.

## How to stop the autopilot
- **Polite stop:** any agent emits `STOP: <reason>` → loop halts → Telegram ping → you investigate.
- **Hard stop:** revert the Routine on `claude.ai/code/routines`.
- **Nuclear stop:** revert the `chore/autopilot-setup` PR; the deny rules in `.claude/settings.json` go away with it.

## Where to look when something fails
- A workflow file → `.github/workflows/{ci,post-deploy}.yml`
- A hook denied a tool call → `.claude/hooks/*.sh` (block-dangerous, sql-guard, protect-paths)
- A test failed locally → `babun-crm/apps/web/e2e/` (Playwright report at `babun-crm/apps/web/playwright-report/index.html`)
- An RLS probe failed → it's the canary; consult `RLS_AUDIT.md` and re-run the probe against staging before touching prod.
- Telegram is silent → `TG_BOT_TOKEN` and `TG_CHAT_ID` env vars on the Routine, plus webhook URL secret in GitHub Actions.

## Scoring rubric (max 20)

- **Functionality (0–5):** +1 if `error.tsx` present, +1 if `loading.tsx` present, +2 if any e2e test exists for this route, +1 baseline.
- **UI / UX (0–5):** start at 5, −1 if hex colors literal in JSX (≥ 1), −1 if hardcoded RU literals > 5 (no dictionary), −1 if file > 400 lines (likely cramped DX), −1 if no `loading.tsx`.
- **Performance (0–5):** start at 5, −1 if > 400 lines, −2 more if > 1000 lines (probably needs split + virtualization), −1 if the route lists > 50 entities and has no `useVirtualizer`.
- **Security (0–5):** start at 5, −1 if `any`/`as unknown as` usages > 0 (type erosion), −1 if route writes to Supabase without obvious Zod validation (heuristic — verified per-story by security-auditor).

`(20 − total) * log(sentry_users + 2)` ranks the backlog. Lower total = higher priority.

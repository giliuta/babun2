# Babun — глоссарий терминов

Единый словарь сущностей и слов, которыми мы говорим внутри продукта
(UI на русском) и в коде (английский). Перед добавлением нового
понятия — сверьтесь сюда. Перед переименованием существующего — здесь
же фиксируйте новый канон и причину.

Это живой документ. Источник истины для §1.1 audit-fixes плана.

---

## Доменные сущности

| RU (UI)        | EN (код)        | Что это                                                                       |
|----------------|-----------------|-------------------------------------------------------------------------------|
| Запись         | `appointment`   | Визит клиента к сотруднику или бригаде. **НЕ** «визит», **НЕ** «событие».     |
| Событие        | `event` / `personal_event` | Личная активность в календаре: обед, отпуск, выезд в офис, заметка. Не привязано к клиенту. |
| Сотрудник      | `master` *(legacy)* / `staff` *(новые поля)* | Универсальная роль работника. На UI везде «Сотрудник», в коде поле `master_id` сохранено для совместимости с миграциями. |
| Бригада / Команда | `team`       | Группа сотрудников. С v510 UI говорит «Команда»; код и URL `/dashboard/teams` остались на `team`. Старое «Бригада» в комментариях допустимо. |
| Клиент         | `client`        | Покупатель услуги. Хранит контакты, объекты, заметки.                         |
| Услуга         | `service`       | Каталог: название, цена, длительность, цвет, привязка к командам.             |
| Возвраты       | `recurring_reminder` | Повторное касание клиента (раз в N дней / выбранная дата). С v518 UI говорит «Возвраты», URL `/dashboard/recurring` сохранён. **НЕ** «Напоминания». |
| Заметки клиента | `client.comment` *(legacy column)* | Длинный текст: «аллергии, предпочтения, особенности». С v522 UI говорит «Заметки», БД-поле осталось `comment`. |
| Комментарий записи | `appointment.comment` | Одноразовая заметка к визиту. **Не** заметки клиента — отдельное поле. |
| Объект клиента | `client_location` / `location` | Физический адрес + опционально оборудование. У одного клиента может быть несколько. |
| Доступ         | `permission` / `permissions` | Per-сотрудник права (31 флаг, сгруппированы по разделам). |
| Тариф          | `tier` / `plan` | Подписка тенанта. Свободный, Базовый, Бизнес. |
| Тенант         | `tenant`        | Один бизнес-аккаунт. Изолированный набор клиентов / команд / записей. |

## Роли (`MasterRole`)

| RU            | EN           | Что может                                                                  |
|---------------|--------------|----------------------------------------------------------------------------|
| Администратор | `admin`      | Всё, включая настройки тенанта и финансы.                                  |
| Диспетчер     | `dispatcher` | Записи, клиенты, чаты, финансы команды. Без управления компанией.          |
| Старший       | `lead`       | Бригадир. Свои записи, своя бригада. С v510 — «Старший» (раньше «Бригадир»). |
| Помощник      | `helper`     | Минимум: только свои назначенные записи.                                   |

## Тон и копирайтинг

- **На «вы».** «Создайте», «Выберите», «Добавьте». **Никаких** «тапни / свайпни / выбери» в десктоп-копи. Мобильные подсказки в install-prompts допустимы.
- **Кнопки.** Единые тексты: «Сохранить» / «Отмена» / «Удалить» / «Создать» / «Добавить» / **«Далее»** (не «Дальше»).
- **Подсказки взаимодействий.** Зависят от устройства:
  - Desktop: «Клик — открыть. Правый клик — меню. Потяните за ручку ☰ — переместить.»
  - Mobile: «Нажмите — открыть. Свайп вправо — в архив. Свайп влево — удалить. Долгое нажатие — меню.»
- **Деструктивные действия в диалогах.** Primary — «Не сохранять / Удалить» (красная заливка), secondary — «Сохранить» (только когда форма валидна).

## Что НЕ называть так

| ❌ Не используйте | ✅ Замена |
|---|---|
| Напоминания | Возвраты |
| Мастер (в UI) | Сотрудник |
| Бригадир | Старший |
| Дальше (кнопка) | Далее |
| Тапни / Свайпни (на desktop) | Нажмите / Клик / Правый клик |
| Визит | Запись |
| Комментарий о клиенте | Заметки о клиенте |

## История ключевых переименований

| Версия | Из | В | Причина |
|---|---|---|---|
| v510 | «Бригада» (UI) | «Команда» | SaaS-нейтральный термин: подходит и салону, и фотографу, не только сервису-выездному. URL и БД-поле `team_id` остались. |
| v510 | «Бригадир» (MasterRole label) | «Старший» | Чтобы не привязывать роль к слову «бригада». |
| v518 | «Напоминания» | «Возвраты» | Точнее описывает функцию (повторное касание), не путается с push-уведомлениями. |
| v518 | «Мастера» (UI заголовки) | «Сотрудники» | Универсальная общая роль. |
| v518 | «Дальше» | «Далее» | Формальный тон. |
| v522 | «Комментарий» (клиент) | «Заметки» | Семантически точнее для freeform-блокнота. Поле БД осталось `comment`. |

## Правила нейминга в коде

- **EN в коде, RU в UI** (см. CLAUDE.md). Все идентификаторы, имена функций, переменных, тип-ключей — на английском.
- **Файлы**: kebab-case для модулей в `lib/`, PascalCase для React-компонентов.
- **Тест-айдишники**: `data-testid="<surface>-<element>"`, kebab-case (`sidebar-nav-clients`, `appointment-sheet-save`).
- **Маркеры версий**: каждый коммит, меняющий UX, бампает `BUILD_VERSION` в `packages/shared/src/common/utils/version.ts` со слагом-описанием. UI рендерит `DISPLAY_VERSION` (формула `v1.<minor>.<patch>` из v513).

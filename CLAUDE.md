## Identity
CRM + скоро SaaS для сервисных бизнесов. Первый клиент — **AirFix** (кондиционеры, Кипр, 2 бригады, 903+ клиентов). В будущем продаём как платформу другим сервисам.

## Stack (LOCKED — не менять без явного запроса)
- **Framework:** Next.js **16** App Router + Turbopack (breaking changes vs 14 — см. `babun-crm/apps/web/AGENTS.md`)
- **Monorepo:** Turborepo with npm workspaces (`babun-crm/apps/web`, `babun-crm/apps/mobile`, `babun-crm/packages/shared`)
- **Language:** TypeScript strict mode
- **Styling:** Tailwind CSS **v4** (не v3)
- **UI:** shadcn-style custom components (не устанавливаем npm-пакет)
- **DB сейчас:** localStorage (prototype phase) → Supabase (project `rdtokosbqvgemicqeqwz`, eu-west-1)
- **Multi-tenancy:** RLS via `current_tenant_id()` SECURITY DEFINER reading `app_metadata.tenant_id` from the JWT — never `user_metadata`
- **Deploy:** Vercel (branch `master` → auto-deploy) at https://babun.app
- **Repo:** github.com/giliuta/babun2 — branch **`master`** (не `main`)
- **PWA:** service worker `babun-v{N}`, auto-update через `ServiceWorkerRegister`

## Golden Rules (MUST — нарушение = откат)
1. **НИКОГДА** не удаляй и не перемещай `babun-crm/apps/web/src/app/`
2. **ВСЕГДА** `npx tsc --noEmit` после серии правок в одной фиче (не обязательно после каждого файла — наш tsc медленный)
3. **ВСЕГДА** bump `BUILD_TAG` в `app/dashboard/page.tsx` и `CACHE_VERSION` в `public/sw.js` при изменении UI — чтобы пользователь видел, что новая версия активна
4. **НИКОГДА** не трогай `ServiceWorkerRegister.tsx` без явного запроса — там тонкий dev/prod разрыв
5. **Manual workflow:** push в `master` после завершения фичи (Vercel deploys from master). **Autopilot workflow:** push в `story/NNN-<slug>`, PR, merge after green gates. Never push directly to master from autopilot.
6. **НИКОГДА** не используй `any` — если TypeScript ругается, разбирайся с типами, а не обходи
7. **Максимум 400 строк** на компонент — если больше, разбивай на sub-components
8. **RU в UI, EN в коде.** Переменные, функции, комментарии — только английский
9. **Один логический коммит = одно сообщение.** Не меняй 10 несвязанных файлов в один commit
10. **НИКОГДА** не ставь хуки в `.claude/settings.json` которые запускают `tsc` на каждую правку — это всё убьёт

## Tenancy (autopilot contract)
- Tenant id lives in JWT `app_metadata.tenant_id`. Never `user_metadata`.
- Every table with tenant scope has `tenant_id uuid not null references public.tenants(id) on delete cascade` + RLS using `(select public.current_tenant_id())` inside USING / WITH CHECK clauses (initPlan caching). `current_tenant_id()` returns `uuid` (not `text` — see STORY-038 migration and `docs/RLS_AUDIT.md`).
- Every new table requires a cross-tenant Playwright probe in `babun-crm/apps/web/e2e/security/rls/`.
- service_role bypass policies must be explicit (`for all to service_role using(true) with check(true)`) — legacy auto-bypass is off after the JWT-Signing-Keys migration.

## Workflow — two modes

### Manual mode (existing)
The classic Plan-then-Code loop you've been running:
1. Прочитай `docs/architecture.md` и `docs/coding-patterns.md`
2. Прочитай актуальный `docs/roadmap.md`
3. Создай `docs/stories/STORY-NNN.md` через `/plan {feature}`
4. **НЕ ПИШИ КОД пока план не записан.** Показываем план — ждём «ок».
5. Реализация: миграции → types → lib → API → components → UI
6. `npx tsc --noEmit` после основных изменений
7. Если UI: bump `BUILD_TAG` + `CACHE_VERSION`
8. `git push origin master`

### Autopilot mode (new)
For unattended, end-to-end execution:
1. `/full-pipeline-autopilot --max-pages=N` (default 1) — drives strategist → architect → developer → tester → designer → security-auditor → performance-optimizer per page.
2. Branch is **always** `story/NNN-<slug>` off latest master. **Never** push directly to master.
3. PR is opened automatically. Vercel preview triggers `.github/workflows/ci.yml`. Playwright + axe-core run against the preview URL.
4. `security-auditor` is the mandatory pre-merge reviewer (no Claude Code GitHub Action because there's no Anthropic API key).
5. After merge, `.github/workflows/post-deploy.yml` runs smoke tests against production; on failure it auto-reverts and pings Telegram.
6. Cloud Routine: `/schedule daily babun autopilot at 06:00 Europe/Nicosia` runs the loop with `--max-pages=3`.

## Context Management
- Новая большая фича → новая сессия с `/clear`
- Перед `/clear` — сохрани прогресс в соответствующую `STORY-NNN.md`
- При возврате к проекту: прочитай CLAUDE.md → docs/roadmap.md → текущую STORY → `git log --oneline -5`

## Critical Known Issues (must not regress)
- **iOS Safari pinch-zoom** на календаре работает только с `userScalable: false` + gesture events (см. `app/dashboard/page.tsx`). НЕ возвращай `userScalable: true`, иначе zoom на календаре сломается.
- **`touchAction: "pan-y"`** на outer scroller нужен для forwarding pinch в JS. Не меняй на `none` или `auto`.
- **`SwipeableCalendar`** имеет собственный touch-handler, отменяющий swipe при 2+ пальцах — не удаляй этот guard.
- **Seed appointments** из `MOCK_APPOINTMENTS` имеют `client_id: null` — клиентские имена хранятся в `comment` как fallback. `AppointmentBlock` это учитывает.
- **AirFix seed leak (STORY-053a)** — fresh tenants не должны видеть AirFix данные. `LEGACY_LOCAL_KEYS` cleanup и `DEFAULT_MASTERS=[]` / `DEFAULT_TEMPLATES=[]` обязательны. security-auditor проверяет любые регрессии.

## Hard stops (autopilot must ASK the user, not act)
1. Any change to `current_tenant_id()` or RLS infrastructure
2. Adding a third-party dependency with > 1 MB minified size
3. Bumping the Next.js or Supabase major version
4. Touching `babun-crm/apps/web/src/middleware.ts`, `supabase/config.toml`, or `.github/workflows/*`
5. Editing `.env*` (denied by hook + permissions)
6. Force pushes; pushing directly to master from autopilot mode
7. Any cross-tenant test newly failing on master
8. Sentry P0 affecting > 50 users in 1 hour
9. Weekly cost forecast > $50 (Max 20× plan ceiling)

These are enforced by `.claude/hooks/{block-dangerous,protect-paths,sql-guard}.sh` plus `.claude/settings.json` `permissions.deny`.

## Commands

### Existing (manual mode)
- `/plan {feature}` — план новой фичи → `docs/stories/STORY-NNN.md`
- `/implement {story-id}` — реализация story по плану
- `/test` — `tsc + eslint` (тестов пока нет вне vitest)
- `/review` — анализ `git diff master`
- `/status` — dashboard состояния проекта
- `/debug {описание}` — диагностика бага
- `/setup` — проверка окружения
- `/full-pipeline {arg}` — существующий 9-фазовый pipeline через chrome-devtools

### New (autopilot mode)
- `/audit-all-pages` — генерирует/обновляет `docs/BACKLOG.md` с 4-axis скором
- `/full-pipeline-autopilot [--max-pages=N] [--page=/route]` — state-machine loop
- `/polish-page <route>` — designer + perf + security проход без рефакторинга
- `/fix-all-bugs` — заворачивает топ Sentry issues в PR
- `/production-ready` — финальный go/no-go гейт

## Agents
- **strategist** (opus) — backlog scoring, story dispatcher
- **architect** (opus) — data model, RLS deltas, file plan ≤ 400 lines/file, ADRs
- **developer** (sonnet) — реализация, один коммит = одна причина
- **tester** (sonnet) — Vitest + Playwright + axe-core + cross-tenant RLS probes
- **designer** (opus) — chrome-devtools polish + iOS HIG checklist
- **security-auditor** (opus) — **MANDATORY pre-merge reviewer** (no claude-review.yml)
- **performance-optimizer** (sonnet) — Lighthouse, bundle, EXPLAIN ANALYZE

Plus 20+ Babun-specific specialist agents under `.claude/agents/babun-*.md` for narrow domains (calendar, brigades, clients, finance, …).
